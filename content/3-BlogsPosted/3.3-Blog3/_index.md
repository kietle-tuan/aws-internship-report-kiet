# AWS STOCK ANALYZER PROJECT OVERVIEW

## 1. Overview

This blog introduces the final project **AWS Stock Analyzer**.

The project was selected by my team during the internship period after learning basic AWS Cloud Computing concepts. The purpose of the project is to build a simple web-based application that allows users to enter a stock symbol and receive related stock analysis information.

In this project, my role was **QA Tester**. I focused on checking application behavior, testing basic functions, and recording issues instead of independently deploying the whole AWS infrastructure.

---

## 2. Project Purpose

The main purpose of **AWS Stock Analyzer** is to help users review stock-related information through a simple web application.

The project is designed for academic and internship purposes. It helps the team practice:

- Basic web application structure.
- Frontend and backend connection.
- Cloud-based application concepts.
- Basic data processing flow.
- Manual testing and project review.

This project is not a professional financial trading system. It is mainly used to apply what we learned during the AWS Cloud Computing internship.

---

## 3. Basic Application Flow

The basic workflow of the application is:

1. The user enters a stock symbol on the web interface.
2. The frontend sends the request to the backend.
3. The backend receives and processes the request.
4. The system retrieves or prepares stock-related information.
5. The backend returns the result.
6. The frontend displays the result to the user.

This flow helped me understand how different parts of a web application work together.

---

## 4. Main Components

| Component | Description |
|---|---|
| Frontend | Provides the user interface for entering stock symbols and viewing results |
| Backend | Handles user requests and processes stock-related logic |
| Data/API Source | Provides stock-related data used by the application |
| AWS Cloud Services | Support the project through hosting, processing, storage, or monitoring depending on the setup |
| QA Testing | Checks whether the application works correctly from the user perspective |

---

## 5. Expected Features

The expected features of the AWS Stock Analyzer project include:

- A simple web interface.
- A stock symbol input field.
- A search or submit button.
- A result display section.
- Basic backend response.
- Basic error handling for invalid input.
- Frontend-backend communication.

These features are suitable for a beginner-level cloud application project.

---

## 6. My Role as QA Tester

My role in the project was **QA Tester**.

My main responsibilities included:

- Understanding the project requirements.
- Reviewing the expected features.
- Preparing basic manual test cases.
- Testing the frontend interface.
- Testing valid and invalid stock symbol inputs.
- Checking backend response behavior.
- Checking frontend-backend integration.
- Recording testing observations.
- Sharing feedback with the team.

My contribution focused on testing and review, not full system deployment.

---

## 7. Testing Direction

For this project, I focused on basic manual testing.

The main testing areas included:

| Testing Area | Description |
|---|---|
| Frontend Testing | Check layout, input field, button, and result display |
| Input Validation | Test valid input, empty input, and invalid stock symbols |
| Backend Testing | Observe basic request and response behavior |
| Integration Testing | Check whether frontend and backend work together properly |
| Usability Checking | Check whether the application is easy to understand and use |

This testing direction helped me check the project from the user perspective.

---

## 8. What I Learned From the Project

Through the AWS Stock Analyzer project, I learned that a web application includes many parts that need to work together.

I understood more about:

- How frontend and backend communicate.
- Why input validation is important.
- How backend responses affect the user experience.
- Why testing should be done carefully before final submission.
- How QA feedback can help improve a project.

Although my testing was still basic and manual, it helped me understand the responsibility of a QA Tester in a software project.

---

## 9. Difficulties

During the project, I faced some difficulties:

- I needed time to understand the project flow.
- Some backend responses were not easy to understand at first.
- I had to distinguish between frontend issues and backend issues.
- I still had limited experience with AWS deployment.
- My testing skill was still at a beginner level.

Because of these difficulties, I focused on testing slowly and recording observations clearly.

---

## 10. Reflection

The AWS Stock Analyzer project helped me connect AWS learning with a real project context.

From this project, I realized that learning AWS is not only about knowing individual services, but also understanding how services can support an application.

As a QA Tester, I also learned that testing plays an important role in improving software quality. Even simple manual testing can help identify issues and make the application more reliable.
