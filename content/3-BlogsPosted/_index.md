## 3. Blog 1: Introduction to AWS Cloud Computing

### Summary

This blog introduces the basic idea of cloud computing and AWS.

Cloud computing allows users and organizations to use computing resources such as servers, storage, networking, and databases through the internet instead of managing physical infrastructure directly.

AWS provides many cloud services that can support application development, deployment, storage, monitoring, and security.

### What I Learned

Through this topic, I learned:

- Cloud computing helps reduce the need to manage physical servers.
- AWS provides many services for different parts of an application.
- Basic AWS knowledge is important before working on a cloud-based project.
- IAM, EC2, S3, and CloudWatch are common services for beginners to study.

---

## 4. Blog 2: Basic AWS Services

### Summary

This blog summarizes several AWS services that I learned during the internship.

The main services include:

| AWS Service | Basic Purpose |
|---|---|
| IAM | Manage users, roles, and permissions |
| EC2 | Provide virtual servers for running applications |
| S3 | Store files and objects |
| VPC | Create and manage network environments |
| Security Groups | Control inbound and outbound traffic |
| CloudWatch | Monitor logs, metrics, and system behavior |

### What I Learned

I learned that each AWS service has a specific role in a cloud system.

For example, EC2 can be used for computing resources, S3 can be used for storage, and CloudWatch can be used to observe system behavior.

At this stage, my understanding was still basic, but these services helped me understand how a cloud application can be organized.

## 5. Blog 3: AWS Stock Analyzer Project Overview

### Summary

This blog introduces the final project **AWS Stock Analyzer**.

The project is a web-based application that allows users to enter a stock symbol and receive stock-related analysis information.

The basic project flow is:

1. The user enters a stock symbol.
2. The frontend sends the request to the backend.
3. The backend processes the request.
4. The system returns stock-related information.
5. The frontend displays the result to the user.

### What I Learned

Through this project, I learned more about how frontend, backend, and cloud services can work together in an application.

I also understood that a project does not only require development, but also testing and review to make sure the main functions work correctly.

## 6. Blog 4: QA Testing in AWS Stock Analyzer

### Summary

This blog explains my role as a **QA Tester** in the AWS Stock Analyzer project.

My testing work focused on basic manual testing, including frontend checking, backend response checking, input validation, and frontend-backend integration.

### Testing Areas

| Testing Area | Description |
|---|---|
| Frontend Testing | Check layout, input field, button, and result display |
| Input Validation | Test valid input, empty input, and invalid stock symbols |
| Backend Testing | Observe API request and response behavior |
| Integration Testing | Check whether frontend and backend work together |
| Usability Checking | Check whether the application is understandable for users |

### What I Learned

I learned that testing helps identify problems before the final project review.

As a QA Tester, I needed to check the application from the user perspective and record issues clearly so the team could improve the project.
