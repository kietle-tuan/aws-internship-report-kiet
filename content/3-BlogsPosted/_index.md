# POSTED BLOGS

## 1. Overview

During the internship, I prepared several short blog-style learning notes related to **AWS Cloud Computing**.

These posts were used to summarize what I learned from AWS Study Group materials, YouTube tutorials, and project practice. The purpose was to help me review basic AWS concepts and connect them with the final project **AWS Stock Analyzer**.

The blog contents are written at a basic level and reflect my actual learning process.

## 3. Blog Summary

### Blog 1: Introduction to AWS Cloud Computing

This blog introduces the basic idea of cloud computing and AWS.

It explains how cloud computing allows users to use computing resources such as servers, storage, networking, and monitoring through the internet instead of managing physical infrastructure directly.

Through this topic, I learned that AWS provides many cloud services that can support web application development, deployment, storage, security, and monitoring.

---

### Blog 2: Basic AWS Services

This blog summarizes several AWS services that I learned during the internship.

The main services include:

- IAM
- EC2
- S3
- VPC
- Security Groups
- CloudWatch

This blog helped me review the basic purpose of each service and understand how these services can support a cloud-based application.

---

### Blog 3: AWS Stock Analyzer Project Overview

This blog introduces the final project **AWS Stock Analyzer**.

The project is a simple web-based application that allows users to enter a stock symbol and receive stock-related analysis information.

In this project, my role was **QA Tester**. I focused on checking application behavior, testing basic functions, and recording issues instead of independently deploying the whole AWS infrastructure.

## 4. Learning Reflection

Preparing these blog-style notes helped me summarize my learning process during the internship.

Although these posts are not advanced technical articles, they reflect my real progress from learning AWS basics to participating in the final project.

The blogs also helped me understand the connection between AWS theory, cloud application structure, and QA testing activities.
