# SELF-ASSESSMENT

## 1. Overview

This section presents my self-assessment after completing the AWS Cloud Computing internship period.

During this internship, I learned basic AWS concepts through tutorials, AWS Study Group materials, YouTube videos, and project practice. In the final project **AWS Stock Analyzer**, I worked as a **QA Tester**.

My self-assessment is based on my actual learning process and project contribution. I do not consider myself an advanced AWS user yet, but I have gained basic knowledge of cloud computing and practical experience in manual testing.

---

## 2. Learning Progress

At the beginning of the internship, my understanding of AWS Cloud Computing was still limited.

During the first weeks, I learned basic AWS services such as IAM, EC2, S3, VPC, Security Groups, and CloudWatch. Most of my learning was based on guided tutorials and online materials.

From Week 7, my team selected **AWS Stock Analyzer** as the final project, and I took the role of **QA Tester**. My work focused on checking the application behavior, testing basic functions, and recording testing observations.

---

## 3. Knowledge Gained

Through the internship, I gained basic knowledge in the following areas:

| Area | What I Learned |
|---|---|
| Cloud Computing | Understood the basic idea of using cloud services instead of physical infrastructure |
| AWS Services | Learned the basic purpose of IAM, EC2, S3, VPC, Security Groups, and CloudWatch |
| Web Application Structure | Understood the relationship between frontend, backend, and cloud services |
| QA Testing | Learned how to check application behavior from the user perspective |
| Manual Testing | Practiced testing input cases, frontend display, backend response, and integration flow |
| Project Collaboration | Learned how to discuss issues and share testing feedback with team members |

---

## 4. My Contribution

In the **AWS Stock Analyzer** project, my main contribution was related to QA testing.

My tasks included:

- Understanding the project requirements.
- Reviewing the main features of the application.
- Preparing basic manual test cases.
- Testing the frontend interface.
- Testing valid and invalid stock symbol inputs.
- Checking backend response behavior.
- Checking frontend-backend integration.
- Recording testing results and observations.
- Giving feedback to support project improvement.

My role was focused on testing and review, not on independently building or deploying the whole AWS infrastructure.

---

## 5. Strengths

During the internship, I think I had some strengths:

- I was willing to learn AWS concepts from basic materials.
- I followed tutorials and reviewed documents to improve my understanding.
- I tried to complete my assigned QA testing role responsibly.
- I recorded testing observations clearly.
- I understood the importance of checking both frontend and backend behavior.
- I worked with the team to support the final project.

---

## 6. Limitations

I also had some limitations during the internship:

- My AWS knowledge was still at a beginner level.
- I did not have much experience with real AWS deployment.
- From Week 3 to Week 5, I mainly learned through YouTube videos and documents instead of doing many hands-on tasks.
- My testing was mainly manual and basic.
- I still need more practice with advanced testing tools, automation testing, and cloud monitoring.
- I needed time to understand how different AWS services connect in a real system.

These limitations show that I still need to continue learning and practicing after the internship.

## 8. Lessons Learned

After completing the internship, I learned that AWS Cloud Computing is a broad field and requires continuous practice.

I also learned that a software project is not only about building functions, but also about testing, reviewing, and improving the application before final submission.

As a QA Tester, I understood that testing helps identify errors and improve user experience. Even basic manual testing can be useful when the tester checks the application carefully and records issues clearly.

---

## 9. Future Improvement Plan

After this internship, I plan to improve in the following areas:

- Review AWS basic services more deeply.
- Practice more hands-on AWS labs.
- Learn more about backend API testing.
- Learn basic automation testing tools.
- Improve my ability to write test cases and testing reports.
- Study how cloud-based applications are deployed and monitored in real projects.

These improvements will help me become more confident in future cloud computing and software testing projects.

---

## 10. Conclusion

Overall, this internship helped me build a basic foundation in AWS Cloud Computing and understand the role of QA testing in a project.

Although I still have limitations, I gained useful knowledge and practical experience through the **AWS Stock Analyzer** project.

My contribution was mainly in manual testing, checking application behavior, and supporting the team with testing feedback. This matches my actual role and learning progress during the internship.
